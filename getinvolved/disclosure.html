Disclosure

We'd like to make clear how this project works and who holds what responsibilities. We want everyone involved to understand how decisions are made and where they can contribute. If you feel this page does not reflect the current state of project ownership and responsibilities, please contact the team on IRC or the Developer Forum.

Stewardship

This project is developed and released by Red Hat with assistance from the Java developer community. The project lead is appointed by Red Hat, and has the power to accept and reject contributions to the project and set the roadmap. Red Hat employees assigned to work on Overlord as well as community contributors, in this regard, all answer to the project lead.

Requirement Definition

The requirements and roadmap for Overlord are driven by Red Hat and by the community. Typically, when a major release is being planned, the project lead will take input from both Red Hat and the community, and evaluate what can be done in the necessary time. This will define what features will be addressed in a given release. The project lead will then publish the roadmap on the User Forum and Issue Tracker (JIRA)

Overlord is supported component of Red Hat JBoss Fuse Service Works (FSW), so many requirements are often driven by FSW. You can download FSW for free from the Red Hat customer portal.

The best way for you to suggest requirements for a future release of Overlord is to enter a feature request in JIRA. Lobbying for features using the voting feature in JIRA, and posting to the User Forum, is a great way to express interest as well. Although, contributing is the single most expedient way to get a capability into a particular release.

Release Cycle

Overlord releases major (new features and API changes), minor (new features but no breaking API changes) and micro releases (patches). Once a version of Overlord is included in a GA FSW release, Overlord moves on to it's next minor version. If you are looking for a long term stream of patches, we recommend using FSW

Contribution

Overlord welcomes, thrives on and greatly values community contribution, and makes the bar to contributing as low as possible. If you have any questions on contributing, check out our guide to contributing, then join us on IRC and speak to a member of the core Overlord team!
